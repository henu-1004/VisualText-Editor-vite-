4.텍스트 수정
   vi에디터와 다르게 키보드를 이용하여 바로 문자를 입력 가능. 
   뉴라인문자, 백스페이스, 라인나누기 등등을 가능. 
   영어만 수정하는 것으로 가정
5.이동
   특정키를 눌러 빠르게 이동가능.
   화살표, 홈, 엔드, 페이지 업, 페이지 다운
6.저장
   파일이 수정되었을 경우. ctrl+s를사용하여 저장 가능. 
   새로운 파일을 저장하려하는 경우 파일이름을 입력해야함.
7.탐색
   ctrl+f를 사용.
   찾고자 하는 문자를 입력하면 처음 찾아진 문자가 하이라이트 된다.
   화살표키를 사용하며 다음이나 이전으로 이동한다.
   enter 키를 누르면 검색을 종료하거나 탐색결과에서 수정이 가능하다.
   esc를 누르면 포기하고 커서는 탐색 이전으로 돌아간다.
8.나가기
   ctrl+q를 눌러 나간다. 
   저장하지 않은 상태로 나가기를 원한다면 ctrl+q를 2번이상 누른다.



ncurses 라이브러리: 터미널 환경에서 텍스트 기반 사용자 인터페이스를 만들기 위한 표준 라이브러리입니다. ncurses를 사용하면 텍스트 에디터의 터미널 버전을 구현할 수 있습니다.

관련 헤더 파일: <ncurses.h>

GTK+ 또는 Qt: 그래픽 사용자 인터페이스 (GUI)를 만들기 위한 강력한 라이브러리 중 하나를 선택할 수 있습니다. 이러한 라이브러리를 사용하면 그래픽 환경에서 텍스트 에디터를 개발할 수 있습니다.

GTK+ 관련 헤더 파일: <gtk/gtk.h>
Qt 관련 헤더 파일: <QtWidgets/QApplication>, <QtWidgets/QMainWindow>, 등
GNU Readline: 명령어 라인에서 사용자 입력을 처리하고 명령어 모드를 구현하는 데 유용한 라이브러리입니다.

관련 헤더 파일: <readline/readline.h>, <readline/history.h>

LibEdit (Editline): GNU Readline과 유사한 명령어 라인 편집 라이브러리입니다. 명령어 모드와 입력 모드를 지원하는 데 사용할 수 있습니다.

관련 헤더 파일: <histedit.h>

PDCurses: 터미널 환경에서 사용자 인터페이스를 구현하는 데 도움이 되는 라이브러리로, ncurses와 유사한 기능을 제공합니다.

관련 헤더 파일: <curses.h>
